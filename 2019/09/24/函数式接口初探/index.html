行为参数化

软件工程中一个众所周知的问题就是不管你做什么，用户的需求肯定会改变。如果我们没有在代码设计之初就考虑到以后可能出现变更的情况，那么在软件的后续迭代中，类似的新的功能增加会带来不一样的挑战，而且长期看来，维护性也会大大降低。

行为参数化就是可以处理频繁需求变更场景的一种软件开发方案。

Lambda表达式

行为参数化设计固然可以提高程序灵活性和可扩展性，但是代码仍然不够简洁，现在开始，我将开始把Java8中的lambda表达式展现出来，介绍lambda表达式的使用场合，感受lambda是如何使代码更简洁、更灵活。

Lambda本身很难定义，但是它有几个显著的特征

  • 匿名：没有一个明确的名称，写得少但是表达的意思丰富
  • 函数：lambda也是函数，不属于某个特定的类，但是和方法有相似的特征，有参数列表、函数主体、返回类型，异常
  • 传递：lambda表达式可以作为参数传递给方法或者存储的变量中
  • 简洁：不需要像匿名类写很多模板代码

我们继续将之前的学生列表作为例子

代码更加简洁，但是做得事情没有变少。根据物质守恒定律，写得少了，就要思考多了。我们需要对这种lambda的编程风格理解透彻，才能在写类似代码的时候做到游刃有余。

Lambda表达式有自己的规范，符合规范才会被当作Lambda表达式来处理。

  • 参数列表
  • 箭头
  • lambda主体

下面给出几个例子

使用案例 Lambda示例
布尔表达式 (List list) -> list.isEmpty()
创建对象 () -> new Apple(10)
消费 (Apple a) -> { System.out.println(a.getKillos());}
映射 (String s) -> s.length()
比较 (Apple a1, Apple a2) -> a1.getKillos().compareTo(a2.getKillos())

函数式接口

所谓函数式接口，是只定义了一个抽象方法的接口。比如已知的

函数式接口的作用是在Lambda表达式中允许直接以内联的形式给函数式接口的抽象方法提供实现，并把整个表达式作为函数式接口的实例。我们可以回想一下匿名类的使用，Lambda表达式在匿名类的基础上使用了函数式接口的特性，使得这个表达更加简洁。

总结

至此，我们已经了解了函数式接口的定义和使用方法，进而学习了Lambda表达式的使用。现在还需要总结一下。

  • 函数式接口就是仅仅声明了一个抽象方法的接口
  • 只有在接受函数式接口的地方才可以使用lambda表达式
  • Lambda表达式允许直接内联，为函数式接口的抽象方法提供实现，并且将整个表达式作为函数式接口的一个实例
  • Java 8自带了一些函数式接口，可以直接拿来使用，当然也可以自定义，需要用到@FunctionalInterface注解
